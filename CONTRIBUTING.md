# Contributing to BucketView

Thank you for helping improve BucketView.

## Development workflow

1. Fork the repository and create a feature branch.
2. Install dependencies with `npm ci`.
3. Run the application with `npm run dev`.
4. Verify production compilation with `npm run build:app`.
5. Keep changes focused and avoid committing generated files, credentials, databases, logs, or local editor state.

## Pull requests

- Explain the problem and the chosen solution.
- Include screenshots for visible UI changes.
- Describe the platforms and storage providers used for testing.
- Call out migrations or compatibility changes explicitly.
- Keep user-facing text and documentation free of organization-specific names, endpoints, and credentials.

## Reporting bugs

Include the operating system, BucketView version, storage provider type, reproduction steps, expected behavior, and relevant sanitized logs. Never include access keys or secret keys.
